Project journey, in order

How the codebase grew from a single FastAPI prototype into the current multi-provider, multi-mode, subscription-gated platform.

Foundation — Q&A on documents
Phase 1
FastAPI backend, React + Vite frontend, single OpenAI model (GPT-4o), FAISS vector store, RAG pipeline with text-embedding-3-small.
Outcome: students upload PDFs/DOCX/TXT and get cited answers.
Accounts, sessions, deployment
Phase 2
MySQL 8 user accounts, bcrypt passwords, bearer-token sessions, Docker Compose + AWS EC2 + nginx, phpMyAdmin for ops.
Outcome: real users, persistent chat history, production deploy.
Payments & tiers
Phase 3
Stripe Checkout + Customer Portal + webhooks, Free / Regular / Advanced tiers, monthly quotas for chat, uploads, and voice.
Outcome: monetisation, usage governance, upgrade path.
Auth providers & safety
Phase 4
Google / Facebook / Microsoft Sign-In, Cloudflare Turnstile CAPTCHA, slowapi rate limiting, password reset emails, IDOR hardening, file-size and MIME guards.
Outcome: lower signup friction, abuse protection.
Admin dashboard
Phase 5
Card-based dashboard with KPI strip, charts (signups, MRR, tier mix), user table with search & ban, AdGuard-proof CSS class names (.adm-*).
Outcome: ops visibility without a third-party BI tool.
Visual identity
Phase 6
Sticky frosted-glass navbar, hero with cyberpunk companion video on solid #4E3D8C, animated counters, conveyor-belt "How It Works" cards, bento-grid features, mouse-fleeing dot motif on auth.
Outcome: a marketing front page that actually converts.
Voice input
Phase 7
Browser MediaRecorder → multipart upload → OpenAI Whisper → transcript fills the chat box. Per-tier monthly transcription quota.
Outcome: hands-free Q&A, accessibility, faster long prompts.
Multi-provider LLM
Phase 8
Provider-agnostic registry (OpenAI / Anthropic / Google), tier-gated picker, graceful degradation when keys are missing, Cursor-style dropdown with locked-state hints.
Outcome: students pick the right brain for the job.
Notifications & reminders
Phase 9
APScheduler cron (Mon + Thu 09:00 UTC), HTML email + in-app bell with unread badge, opt-out preferences, idempotent multi-worker lock.
Outcome: re-engagement and upgrade nudges without spam.
Test & Research modes + clearer trust signals
Phase 10
Two new AI modes (quiz me / research synthesis), revamped confidence heuristic with bail-phrase detection, page-level citations, X icon, refreshed feature bento.
Outcome: turns a chatbot into a study + research toolkit.

How tier gating works
The catalog in app/services/llm/registry.py marks every model with a min_tier and a provider key. The /api/models endpoint returns the full list to every user with available + locked_reason, so locked models stay visible (greyed out) and act as built-in upsells.

FAISS (vector state)

One FAISS index per user under backend/faiss_store/, persisted via the faiss_data Docker volume so uploads survive container restarts.

Stores 1536-dim embeddings of every uploaded chunk, with the source filename, page number, and original passage attached as metadata so citations point at "page X" instead of "chunk 42".

File storage

Raw uploads live alongside the FAISS volume, isolated per user so document_service.py can rebuild the index on demand.

Pipelines

Each major user action is implemented as a documented pipeline with guard rails.

RAG query pipeline
request → auth check → quota check → mode dispatch → embed query → FAISS top-K → build prompt with grounding → call selected LLM via registry → calculate confidence → format page-level citations → save to chat_messages → respond.
Benefit: low-hallucination answers with the page they came from.
Document upload pipeline
upload → MIME + size guard (MAX_UPLOAD_BYTES) → loader (pypdf / docx2txt / text) → chunk (CHUNK_SIZE / CHUNK_OVERLAP) → embed (text-embedding-3-small) → write to per-user FAISS index → bump quota counter.
Benefit: students can drop in lecture notes and start asking.
Voice input pipeline
MediaRecorder blob → multipart POST /api/voice/transcribe → MAX_AUDIO_BYTES guard → MIME whitelist → AsyncOpenAI Whisper → transcript JSON → fills the chat input → quota counter bump.
Benefit: hands-free, accessibility-friendly Q&A.
Auth pipeline
signup form → Turnstile verify → bcrypt hash → users row → bearer token → session row → response. OAuth variant: ID-token verify → upsert user → same session issuance.
Benefit: low-friction signup, abuse protection, single token everywhere.
Payment pipeline
upgrade click → backend creates Stripe Checkout session → redirect → Stripe webhook → update tier on users row → reset quotas. Customer Portal link for self-service downgrade / cancel / payment update.
Benefit: PCI-safe upgrades, no card data ever hits our servers.
Reminder pipeline
APScheduler cron (Mon + Thu 09:00 UTC) → claim_run() on reminder_runs → paginate users → respect notification_preferences → SMTP HTML email + insert into notifications → mark_run_finished.
Benefit: gentle re-engagement + Advanced upsell, never duplicated.

Bottom line
From a single-model RAG demo, Laboracle has grown into a 4-service Docker deployment with 5 LLMs behind one registry, voice input, four AI modes, three OAuth providers, Stripe subscriptions, an admin console, and a notification system. Every layer was added behind a feature flag or graceful-degradation guard, so the app keeps running even when individual provider keys are missing.
